# /mermaid-lint -- Mermaid 图审查与优化

扫描文档中的 Mermaid 图，逐个评估可读性，输出问题清单并自动修复。用 `/mermaid-lint` 启动。可指定文件：`/mermaid-lint docs/04_explanation/arch-03*.md`。

## 触发条件

用户输入 `/mermaid-lint`，或要求「检查 mermaid」「优化图表」「图表排版」。

## 执行流程

```
1. 定位目标
   IF 用户指定文件 → 仅扫描该文件
   ELSE → 扫描 $PROJECT_ROOT/docs/ 下所有 .md 文件

2. 提取所有 ```mermaid 块
   记录：文件路径、起始行号、图表类型（flowchart/sequence/state/class/gantt）

3. 逐图评估（见「检查规则」）
   每个图表输出：pass / warn / fail + 具体问题

4. 汇总报告
   按严重度排序，输出表格：文件 | 行号 | 图表类型 | 问题 | 建议

5. 自动修复（需用户确认）
   对所有 fail 和 warn 项，提出修复方案
   用户确认后批量 Edit
```

## 检查规则

### 项目约束（来自 $PROJECT_ROOT/CLAUDE.md，硬规则 -- fail）

| ID | 规则 | 适用类型 |
|----|------|---------|
| P1 | 每文档 Mermaid 图 <= 3 个 | 全部 |
| P2 | 序列图参与者 <= 6 个，步骤 <= 15 个 | sequenceDiagram |
| P3 | 流程图节点 <= 15 个 | flowchart / graph |
| P4 | 类图类 <= 8 个 | classDiagram |
| P5 | 换行使用 `<br>`，禁止 `\n` | 全部 |
| P6 | 每个图表后必须有中文说明段落 | 全部 |

### 可读性规则（软规则 -- warn）

| ID | 规则 | 说明 |
|----|------|------|
| R1 | **长链条需打断**：单方向链 > 8 个节点时必须引入分支或 subgraph | 最重要的规则。10+ 节点的线性链在 TD 方向渲染成高瘦条，LR 方向渲染成宽扁条，字缩到无法阅读 |
| R2 | 方向选择匹配内容结构 | 管线/时序流程用 LR（左到右=时间推进），层级/依赖关系用 TD（上到下=抽象层级） |
| R3 | 超过 8 个节点的流程图使用 subgraph 分组 | subgraph 创造二维布局：TD 方向的图中 subgraph 内部横向排列，打破纵向堆叠 |
| R4 | 节点标签 <= 20 个中文字符 | 过长标签挤压布局 |
| R5 | 连线标签 <= 10 个字符 | 长标签遮挡连线，移到节点内或用注释 |
| R6 | 避免在节点 ID 中使用中文 | 部分渲染器对中文 ID 支持不稳定，用英文 ID + 中文标签 |
| R7 | 序列图避免自调用超过 3 层 | `A->>A` 嵌套过深难以阅读 |
| R8 | 状态图状态数 <= 10 个 | 状态过多时拆分为多图或使用复合状态 |

### 长链条打断策略（R1 的具体手法）

当一条直线链超过 8 个节点时，选择以下策略之一：

**策略 A -- 改方向 + 用 subgraph 分段**：将长管线按阶段分成 2-3 个 subgraph，subgraph 在主方向上堆叠，内部节点在交叉方向上排列。例：

```
flowchart LR
    subgraph sg1["初始化阶段"]
        A --> B --> C
    end
    subgraph sg2["更新阶段"]
        D --> E --> F
    end
    subgraph sg3["加载阶段"]
        G --> H --> I
    end
    sg1 --> sg2 --> sg3
```

9 个节点分成 3 组，LR 方向排 3 列，每列内 3 节点。渲染结果近似正方形而非长条。

**策略 B -- 引入并行分支**：如果流程中存在逻辑分支（如条件判断后合并），用分支替代串联，增加图的"宽度"。

**策略 C -- 拆成多个独立图**：超过 15 节点且无法合理分组时，按阶段拆成 2 个图，各图有独立的中文说明段落。

## Gotchas

### `\n` 在 Mermaid 中不换行
Mermaid 的节点标签中 `\n` 被渲染为字面文本而非换行。必须用 `<br>` 替代。这是本项目 $PROJECT_ROOT/CLAUDE.md 的硬性规则，git 历史中已有一次全量修复提交（fa8473b）。

### 中文标签导致布局偏移
Mermaid 的宽度计算基于等宽字符假设。中文字符实际占位约为英文的 1.5-2 倍，导致节点框过窄、文本溢出。应对：控制中文标签长度，必要时用英文缩写 + 括号注释。

### subgraph 标题与节点 ID 冲突
`subgraph` 的标题文本如果与已有节点 ID 重名，会导致渲染错误或节点消失。始终为 subgraph 使用独立的标识符（如 `subgraph sg_server["服务端"]`）。

### flowchart 中的特殊字符
圆括号 `()`、方括号 `[]`、花括号 `{}` 在节点定义中有语法含义。标签中需要这些字符时，必须用引号包裹整个标签：`A["函数名(param)"]`。

### 序列图 loop/alt 块的缩进
Mermaid 序列图的 `loop`、`alt`、`opt` 块虽然不强制缩进，但缩进极大提高源码可读性。修复时统一 4 空格缩进。
