# 研究规划方法论

## 为什么需要研究计划

没有计划的文档提取容易陷入两个极端：要么过于发散（什么都想写，结果什么都没写完），要么过于局部（只写自己熟悉的部分，忽略整体）。研究计划解决以下问题：

- 明确总体覆盖目标，避免遗漏关键系统
- 按优先级分配精力，确保核心内容先完成
- 量化覆盖进度，让每个阶段都有可验收的产出

## 研究计划结构

在 `_meta/research-plan.md` 中维护总体研究计划，结构如下：

```markdown
# {项目名称} 文档研究计划

## 研究域划分

### P0 — 核心必须覆盖

- [ ] 系统 A：[简短描述，预计产出 X 份文档]
- [ ] 系统 B：[简短描述，预计产出 X 份文档]

### P1 — 重要，第二阶段覆盖

- [ ] 系统 C：[简短描述]
- [ ] 系统 D：[简短描述]

### P2 — 补充，有余力时覆盖

- [ ] 系统 E：[简短描述]

## 执行日志

### 阶段一（开始日期 — 结束日期）
目标：完成 P0 核心系统的架构分析
产出：[列出实际完成的文档]
覆盖率变化：0% → X%
```

## 覆盖率追踪

在 `_meta/documentation-gaps.md` 中维护细粒度的覆盖率数据。

**覆盖率表格格式：**

```markdown
## {系统名称}

| 子模块 | 覆盖状态 | 文档链接 | 置信度 | 备注 |
|--------|---------|---------|--------|------|
| 初始化流程 | 已完成 | arch-02-init.md | HIGH | |
| 网络层 | 进行中 | arch-03-network.md | MEDIUM | 缺少重试逻辑 |
| 存储层 | 未开始 | — | — | P1 优先级 |
| 错误处理 | 已跳过 | — | — | 范围外，暂不覆盖 |

覆盖率：2/4 = 50%
```

**覆盖状态枚举：**

- `已完成`：文档已写，覆盖该子模块的主要路径
- `进行中`：文档草稿存在，尚未完成
- `未开始`：计划覆盖，尚未开始
- `已跳过`：明确决定不覆盖，需说明原因
- `待验证`：文档存在但有 LOW 置信度结论需要后续确认

## 研究阶段划分

将整个文档提取工作分为多个阶段，每个阶段有明确的目标和验收标准。

**阶段定义模板：**

```
阶段 {n}：{阶段名称}
目标系统：[列出本阶段覆盖的系统/模块]
预计文档数：X 份
验收标准：
  - 完成 {系统} 的架构分析，覆盖率达到 X%
  - 所有 P0 文档通过质量自评（B 级以上）
  - 覆盖率追踪表已更新
```

## 阶段执行报告

每个阶段完成后，在 `_meta/research-plan.md` 中追加执行报告：

```markdown
### 阶段 {n} 执行报告（日期）

**目标完成情况：**
- [x] 完成 {系统 A} 架构分析（arch-02、arch-03）
- [ ] {系统 B} 网络层分析未完成（移至下一阶段）

**实际产出：**
- `arch-02-init-flow.md`：初始化流程，覆盖率 80%，HIGH 置信度
- `arch-03-network.md`：网络架构，覆盖率 60%，含 3 个 LOW 置信度结论

**覆盖率变化：**
整体：X% → Y%

**下一阶段调整：**
- {系统 B} 优先级从 P1 上调至 P0（原因：涉及核心流程）
- 增加 {系统 E} 到 P2（新发现的子系统）

**待验证项汇总：**
1. `arch-03-network.md`：「重连逻辑在 WiFi 切换时自动触发」[LOW] — 需源码验证
```

## Gap 分析方法

定期（每完成一个阶段）进行 gap 分析，识别文档盲区。

**Gap 分析步骤：**

1. 列出源码中所有主要目录/命名空间
2. 对比 `_meta/documentation-gaps.md` 中的覆盖情况
3. 识别覆盖率低于 30% 的系统
4. 评估这些系统的重要性，决定是补充覆盖还是标记为已知缺口

**Gap 分析产出格式：**

```markdown
## Gap 分析报告（日期）

整体覆盖率：X%（已完成文档 / 计划总文档数）

### 低覆盖区域

| 系统 | 当前覆盖率 | 重要性 | 决策 |
|------|---------|--------|------|
| {系统 X} | 10% | 高 | 加入 P0，下阶段优先覆盖 |
| {系统 Y} | 0% | 低 | 标记为已知缺口，不纳入计划 |

### 已知缺口声明

以下系统明确不在文档覆盖范围内：
- {系统 Z}：原因 [过于底层/已有外部文档/范围外]
```
