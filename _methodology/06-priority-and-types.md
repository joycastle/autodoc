# 优先级判定与类型判定

本文档定义文档任务的优先级判定规则和文档类型判定规则。doc-discover 和 doc-expert 在创建任务时均依据此文件。项目可根据实际情况调整优先级权重和类型映射。

## 优先级判定

| 信号 | 对应优先级 |
|------|-----------|
| 被 3+ 其他模块 import/调用 | P0-P1 |
| 处理用户输入或金钱（支付、交易） | P0 |
| 仅被 1 个模块使用 | P2 |
| 纯内部工具，不影响线上行为 | P2 |
| README 中专门提及的系统 | P1+ |
| 核心业务模块顶级目录 | 至少 P1 |
| 横切主题（3+ 文档涉及） | P1 |
| 类型平衡填补 | P2 |
| 漂移导致的 improve | 原文档优先级 |
| 角色缺口：覆盖度 NONE 且角色占比大 | P1 |
| 角色缺口：覆盖度 NONE 且角色占比小 | P2 |
| 角色缺口：覆盖度 PARTIAL | P2 |
| verify 任务（类型 A/C） | P2 |
| 端到端旅程：涉及金钱 | P0 |
| 端到端旅程：核心业务循环 | P0 |
| 端到端旅程：跨 3+ 模块 | P1 |
| 故障路径：支付/数据相关 | P0 |
| 故障路径：跨服务恢复 | P1 |
| 隐式耦合：数据一致性风险 | P1 |
| 时序依赖：生命周期协调 | P1 |
| 决策考古：跨模块架构决策 | P1 |
| 运维暗知识：容量/限流 | P1 |
| 运维暗知识：错误码清单 | P2 |

## 类型判定

| 源码位置/发现来源 | 文档类型 |
|------------------|---------|
| 后端业务模块 | sys- 或 core- |
| 前端功能模块 | sys- 或 core- |
| API 服务定义 | ref- |
| 工具模块 | sys- 或 how- |
| 部署脚本 | how- |
| 横切主题 | theory- 或 arch- |
| 类型平衡：教程 | tut- |
| 类型平衡：操作指南 | how- |
