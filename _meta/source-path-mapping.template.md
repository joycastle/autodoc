# 源码路径映射表

本文件定义项目源码的目录结构和模块划分规则。autodoc 的各 skill 在扫描源码、追踪调用链时依据此文件确定探索范围。

使用时复制为 `_meta/source-path-mapping.md` 并填入项目实际路径。

## 扫描目录

| 路径 | 模块粒度 | 说明 |
|------|---------|------|
| `src/{backend}/` | 每个子目录是一个业务模块 | 后端业务逻辑 |
| `src/{frontend}/` | 每个子目录是一个功能模块 | 前端功能模块 |
| `src/{api}/` | 每个子目录是一个 API 服务 | API 定义 |
| `src/{infra}/` | 每个子目录是一个基础设施模块 | 基础设施 |
| `src/{ops}/` | 按服务分组 | 部署与运维脚本 |

## 文档 → 源码映射

| 文档 | 源码路径 |
|------|---------|
| sys-01-{name} | `src/{path1}/`, `src/{path2}/` |
| sys-02-{name} | `src/{path}/` |

## 跳过目录

以下目录不需要独立文档：

- `src/{vendor}/` -- 第三方库
- `src/{test}/` -- 测试目录
- `src/{generated}/` -- 自动生成代码

## 编程语言

主要语言：{language}
文件扩展名：`*.{ext}`（用于 find/grep 过滤）
